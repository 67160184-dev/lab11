## ส่วนที่ 1: JSX Syntax & JSX Rules

### คำถาม 1.1: จงอธิบายความแตกต่างระหว่าง class และ className ใน JSX พร้อมตัวอย่างเหตุผลที่ React ต้องการใช้ className

JSX ต้องใช้ className แทน class เพราะ:

- class เป็นคำสงวน (reserved keyword) ใน JavaScript เลยเอามาใช้ตรงๆ ไม่ได้

- JSX แปลง HTML attribute ให้เป็น JavaScript properties ตอน compile

- ตัวอย่าง:

ถูก: `<div className="container">...</div>`
ผิด: `<div class="container">...</div>` (ทำให้เกิด error)

### คำถาม 1.2: จงเขียน Component ที่ใช้ JSX rules ให้ถูกต้องตามตัวอย่าง ให้แสดง Component ที่แก้ไขแล้ว และอธิบายข้อผิดพลาดแต่ละข้อ

- Component ที่แก้ไขแล้ว

```jsx
function ProfileCard() {
  const userName = "สมชาย";
  const age = 25;

  return (
    <div className="card">
      <h3>ชื่อ: {userName}</h3>
      <p>อายุ: {age}</p>
      <img src="avatar.png" />
      <button>คลิกที่นี่</button>
    </div>
  );
}
```

สิ่งที่แก้ไข:

class → className เพราะ class เป็น reserved keyword ใน JavaScript
userName → {userName} เพราะต้องใส่ {} ถ้าจะเอาค่าตัวแปรมาแสดงใน JSX
age → {age} เหตุผลเดียวกัน ไม่งั้นมันจะแสดงคำว่า "age" ออกมาตรงๆ เลย
<img src="avatar.png"> → <img src="avatar.png" /> เพราะ JSX ต้องปิด tag ทุก tag ถ้าไม่มี children ต้องใส่ / ปิดท้าย

## ส่วนที่ 2: Function Components

### คำถาม 2.1: จงอธิบายความสำคัญของการตั้งชื่อ Component ด้วยตัวพิมพ์ใหญ่ (PascalCase) และแสดงตัวอย่างอย่างน้อย 2 ตัวอย่าง

- ใน React การตั้งชื่อ Component ควรใช้ **ตัวพิมพ์ใหญ่ตัวแรก (PascalCase)** เช่น `UserProfile` หรือ `ProductCard` เพราะ React ใช้ตัวพิมพ์ใหญ่ในการแยกว่า element นั้นเป็น **Component ที่ผู้พัฒนาสร้างขึ้น** หรือเป็น **HTML element ปกติ**

ถ้าชื่อขึ้นต้นด้วยตัวพิมพ์เล็ก React จะคิดว่าเป็น HTML tag เช่น `<div>` หรือ `<span>` ทำให้ Component อาจไม่ทำงานตามที่ต้องการ

ดังนั้นการใช้ PascalCase ช่วยให้:

- React แยก Component ออกจาก HTML element ได้
- โค้ดอ่านง่ายและเป็นมาตรฐานของ React

- ตัวอย่างที่ถูกต้อง

```jsx
function UserProfile() {
  return <h2>ข้อมูลผู้ใช้</h2>;
}

function ProductCard() {
  return <div>สินค้า</div>;
}
```

## ส่วนที่ 3: Props

### คำถาม 3.1: จงสร้าง Component ชื่อ ProductCard ที่รับ Props: name, price, inStock พร้อมอธิบายวิธีการส่ง Props จาก Parent Component

- ProductCard Component

```jsx
function ProductCard({ name, price, inStock }) {
  return (
    <div className="product-card">
      <h3>{name}</h3>
      <p>ราคา: {price} บาท</p>
      <p>{inStock ? "มีสินค้า" : "สินค้าหมด"}</p>
    </div>
  );
}
```

- App.jsx - วิธีส่ง Props

```jsx
function App() {
  return (
    <div>
      <ProductCard name="รองเท้า" price={599} inStock={true} />
      <ProductCard name="กระเป๋า" price={1299} inStock={false} />
    </div>
  );
}
```

**อธิบายความแตกต่างของ Props แต่ละประเภท:**

1. string เช่น name="รองเท้า" → ใช้ " " ได้เลย ไม่ต้องใส่ {}
2. number เช่น price={599} → ต้องใส่ {} ถ้าไม่ใส่จะกลายเป็น string "599" แทน
3. boolean เช่น inStock={true} → ต้องใส่ {} เหมือนกัน หรือถ้าจะส่ง true เขียนแค่ inStock อย่างเดียวก็ได้ React จะถือว่าเป็น true ให้เลย

### คำถาม 3.2: Props destructuring คืออะไร? จงแสดงตัวอย่างเปรียบเทียบระหว่าง: วิธีเก่า: function Card(props) { ... } วิธีใหม่: function Card({ title, description }) { ... } และอธิบายว่าทำไมวิธีใหม่ (destructuring) จึงเป็น best practice

- Props destructuringคือการดึงค่าจาก `props` ออกมาใช้งานโดยตรงใน parameter ของ function ทำให้ไม่ต้องเขียน `props.` ทุกครั้งที่ต้องการใช้ค่าใน Component

- วิธีเก่า (ไม่ใช้ destructuring)

```jsx
function Card(props) {
  return (
    <div className="card">
      <h3>{props.title}</h3>
      <p>{props.description}</p>
    </div>
  );
}
```

- วิธีใหม่ (ใช้ destructuring)

```jsx
function Card({ title, description }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <p>{description}</p>
    </div>
  );
}
```

**เหตุผลที่ destructuring เป็น Best Practice**

1. ทำให้โค้ดสั้นและอ่านง่ายขึ้น

2. ลดการเขียน props. ซ้ำ ๆ

3. ทำให้เห็นชัดเจนว่า Component รับ props อะไรบ้าง

4. เป็นรูปแบบที่นิยมใช้ใน React ปัจจุบัน

## ส่วนที่ 4: Lists & .map()

### คำถาม 4.1: จงอธิบายว่า .map() ทำงานอย่างไรในการแสดงรายการข้อมูล พร้อมตัวอย่างโค้ดที่แสดง: 1.การใช้ .map() เพื่อแสดง list ของ Component 2.เหตุผลว่าทำไมต้องใช้ key property 3.ข้อแตกต่างระหว่าง key={index} และ key={item.id}

- .map() คือ JavaScript method ที่วนลูปผ่านทุก element ใน array แล้ว return ค่าใหม่ออกมา React เลยใช้มันเพื่อแปลง array ข้อมูลให้เป็น array ของ JSX แทน

ตัวอย่างการใช้ `.map()` แสดง list ของ Component

```jsx
const movies = [
  { id: 1, title: "Inception" },
  { id: 2, title: "Titanic" },
  { id: 3, title: "Avatar" },
];

function MovieList() {
  return (
    <div>
      {movies.map((movie) => (
        <p key={movie.id}>{movie.title}</p>
      ))}
    </div>
  );
}
```

**เหตุผลที่ต้องใช้ key property**

1. ใน React ต้องกำหนด key ให้กับ element ใน list เพราะ React ใช้ key เพื่อระบุว่า element แต่ละตัวคืออะไร เมื่อมีการ update ข้อมูล React จะสามารถ เปรียบเทียบและ update DOM ได้อย่างมีประสิทธิภาพ

2. ถ้าไม่มี key React จะไม่สามารถติดตาม element ได้อย่างถูกต้อง และอาจทำให้เกิด warning หรือการ render ที่ไม่ถูกต้อง

**ความแตกต่างระหว่าง key={index} และ key={item.id}**

**key={index}**

- ใช้ตำแหน่งของ array เป็น key
- อาจเกิดปัญหาเมื่อมีการเพิ่ม ลบ หรือเรียงข้อมูลใหม่

**key={item.id}**

- ใช้ค่า id ที่ไม่ซ้ำกัน
- เป็นวิธีที่ดีกว่า เพราะ React สามารถติดตาม element ได้ถูกต้อง

### คำถาม 4.2: สร้าง Component ที่ใช้ .map() เพื่อแสดง: 1.รายชื่อนิสิต (array of objects ที่มี id, name, grade) 2.แสดงเป็นตาราง โดยใช้ .map() ในการสร้าง แต่ละแถว 3.อธิบายเหตุผลว่าทำไมต้องใช้ unique ID เป็น key

**ตัวอย่าง Component ที่ใช้ `.map()` เพื่อแสดงรายชื่อนิสิตในรูปแบบตาราง**

```jsx
const students = [
  { id: 1, name: "สมชาย ใจดี", grade: "A" },
  { id: 2, name: "สมหญิง รักเรียน", grade: "B+" },
  { id: 3, name: "มานะ ตั้งใจ", grade: "A+" },
];

function StudentTable() {
  return (
    <table>
      <thead>
        <tr>
          <th>รหัส</th>
          <th>ชื่อ</th>
          <th>เกรด</th>
        </tr>
      </thead>
      <tbody>
        {students.map((student) => (
          <tr key={student.id}>
            <td>{student.id}</td>
            <td>{student.name}</td>
            <td>{student.grade}</td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

**ทำไมต้องใช้ unique ID เป็น key:**

1. React ติดตามได้ถูกต้อง → ถ้าเพิ่ม/ลบ/สลับแถว React รู้ว่าแถวไหนเปลี่ยนแปลงโดยไม่ต้อง render ใหม่ทั้งหมด
2. ป้องกัน bug → ถ้าใช้ index แทน พอสลับลำดับข้อมูล index เปลี่ยน React อาจแสดงข้อมูลผิดแถวได้
3. id ไม่เปลี่ยนแปลง → ไม่ว่าจะเรียงลำดับยังไง id ของนิสิตแต่ละคนก็ยังเป็นค่าเดิมเสมอ

## ส่วนที่ 5: Best Practices & Naming Conventions

### คำถาม 5.1: จงอธิบายและยกตัวอย่างการตั้งชื่อในแต่ละประเภท ทั้งแบบที่ถูก และแบบที่ผิด: 1.Component Names - ตั้งชื่อแบบไหน? 2.Variable Names - ตั้งชื่อแบบไหน? 3.Constants - ตั้งชื่อแบบไหน? 4.File Names - ตั้งชื่อแบบไหน?

1.  Component Names → PascalCase
    **✅ ถูก**
    - function UserCard() {}
    - function NavBar() {}
    - function ProductList() {}

    **❌ ผิด**
    - function userCard() {}
    - function navbar() {}
    - function product_list() {}

2.  Variable Names → camelCase
    **✅ ถูก**
    - const userName = "สมชาย";
    - const isLoggedIn = true;
    - const productPrice = 599;

    **❌ ผิด**
    - const UserName = "สมชาย";
    - const is_logged_in = true;
    - const ProductPrice = 599; 3. Constants → UPPER_SNAKE_CASE

    **✅ ถูก**
    - const MAX_PRICE = 9999;
    - const API_URL = "https://api.example.com";
    - const TAX_RATE = 0.07;

    **❌ ผิด**
    - const maxPrice = 9999;
    - const apiUrl = "https://api.example.com";
    - const taxrate = 0.07;

3.  File Names → PascalCase สำหรับ Component, camelCase สำหรับไฟล์ทั่วไป

    **✅ ถูก**
    - UserCard.jsx → Component
    - ProductList.jsx → Component
    - useAuth.js → Custom Hook
    - utils.js → ไฟล์ทั่วไป

    **❌ ผิด**
    - usercard.jsx
    - productlist.jsx
    - UseAuth.js
    - Utils.js

### คำถาม 5.2: จากโค้ด Component ต่อไปนี้ให้บอก: 1.ส่วนที่ตั้งชื่อถูกต้อง 2.ส่วนที่ตั้งชื่อผิด (ถ้ามี) 3.แก้ไขโค้ดให้เป็น best practice

**ส่วนที่ถูก:**

- isPremium ใช้รูปแบบ camelCase ซึ่งเป็นมาตรฐานของตัวแปรใน JavaScript

- name และ age เป็นชื่อตัวแปรที่อ่านง่ายและเข้าใจได้

**ส่วนที่ผิด:**

1. user_profile

- Component ควรใช้ PascalCase

- ควรเปลี่ยนเป็น UserProfile

2. user_name และ user_age

- ใช้ snake_case ซึ่งไม่ใช่มาตรฐานของ JavaScript

- ควรใช้ camelCase เช่น userName, userAge

3. IS_PREMIUM

- รูปแบบนี้มักใช้กับ constants

- ควรใช้ isPremium
  4.user_data
- ไม่จำเป็นต้องสร้าง object เพิ่ม เพราะสามารถใช้ props ได้โดยตรง

```jsx
function UserProfile({ userName, userAge, isPremium }) {

```
